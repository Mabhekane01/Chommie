- Frontend
- Backend
- Database schema
- Authentication
- Cart
- Checkout
- Order management

- DO NOT redesign, restructure, or restyle existing UI components.

- DO NOT introduce breaking changes.
- Reuse existing architecture, naming conventions, patterns, and data models.
- Extend existing code only; do not replace or rewrite it.
- Prefer using existing database fields and endpoints before proposing new ones.
- Backend logic must remain clean, modular, and scalable.
- Implement changes incrementally and conservatively.
- If existing code is required for accurate integration, explicitly request it before proceeding.

Introduce enterprise-level, Amazon/Takealot-style enhancements that measurably improve:
- User retention and repeat purchases
- Conversion rate
- Trust and platform credibility
- Performance and perceived speed

SCOPE (PRIORITY ORDER):
1. Retention & repeat purchases
2. Trust & credibility
3. Performance & UX optimizations
4. Logistics & delivery experience

DELIVERABLE FORMAT:
For each proposed feature, provide:
1. Feature name
2. Business value (why it matters)
3. Backend impact (models, queries, APIs — reuse-first approach)
4. Frontend integration points (which existing UI/components are extended)
5. Data flow overview
6. Edge cases and performance considerations
7. Minimal, non-invasive code snippets ONLY where strictly necessary

AVOID COMPLETELY:
- Creating new UI layouts or pages
- Large refactors or architectural changes
- Overengineering
- AI/ML unless explicitly requested
- Generic or theoretical e-commerce advice

ASSUME:
- Modern frontend framework (React / Next.js or similar)
- REST or GraphQL backend
- SQL-based database
- Dockerized environment

PROCESS:
- Start by proposing ONLY the top 3 highest-impact features.
- Wait for explicit confirmation before implementing any feature.

MARKET CONTEXT:
Optimize specifically for African / South African e-commerce realities
(connectivity, delivery constraints, trust sensitivity, mobile usage).
